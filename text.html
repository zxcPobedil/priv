🌐 Веб-технологии и PHP

1. Межсайтовый скриптинг (XSS). Способы противодействия

Определение: Атака, при которой злоумышленник внедряет вредоносный скрипт (обычно JavaScript) в веб-страницу, просматриваемую другими пользователями.

Виды:

  • Хранимые (Stored): Код сохраняется в БД (комментарии).
  • Отраженные (Reflected): Код передается в URL/параметре запроса.
  • DOM-модельные: Уязвимость на стороне клиента (в JS-коде).

Способы защиты:

  1. Экранирование/Кодирование выходных данных (HTML Entities).
  2. Content Security Policy (CSP): Заголовок для ограничения источников.
  3. Фильтрация входных данных ("белые списки").
  4. Использование флагов HttpOnly и Secure для cookie.
2. Искусственный интеллект в ИБ: проблемы развития и внедрения

Применение: Обнаружение аномалий (UEBA), анализ вредоносного ПО, автоматизация SOC (SOAR), биометрия.

Проблемы:

  • Ложные срабатывания (False Positives).
  • Состязательные атаки (обман моделей).
  • Непрозрачность работы (Black Box).
  • Зависимость от качества датасетов.
  • Этические вопросы и предвзятость (Bias).
3. Киллчейн (Cyber Kill Chain). Общие правила построения

Этапы атаки (Lockheed Martin):

  1. Разведка (Reconnaissance)
  2. Вооружение (Weaponization)
  3. Доставка (Delivery)
  4. Эксплуатация (Exploitation)
  5. Установка (Installation)
  6. Управление (C2)
  7. Действия по цели (Actions)

Правило: Эшелонированная защита. Прерывание цепочки на любом этапе не даёт атаке завершиться.

4. Язык PHP: выражения, переменные, функции
  • Переменные: Начинаются с $. Динамическая типизация. Суперглобальные массивы ($_GET, $_POST).
  • Выражения: Фрагмент кода, имеющий значение (присваивание, арифметика).
  • Функции: 
    function sum(int $a, int $b): int { 
    return $a + $b; 
}
    Анонимные (замыкания), стрелочные fn().
5. Выражения и управление процессом выполнения в PHP
  • Условия: if...elseif...else, switch/case.
  • Циклы: while, for, foreach.
  • Управление циклом: break, continue.
  • Включения: include, require.
  • Конструкция match (PHP 8+): Строгий аналог switch.
6. Принципы проектирования дизайна веб-приложений
  • UX-принципы: Ориентация на пользователя, простота.
  • Адаптивность: Mobile First, медиа-запросы, Flex/Grid.
  • Доступность (A11y): ARIA-атрибуты, контрастность, семантический HTML.
  • Методологии CSS: БЭМ (Блок-Элемент-Модификатор).
7. Hover-эффекты и способы их реализации

Hover-эффект: Изменение стиля при наведении мыши.

  1. CSS (:hover): Изменение цвета, тени, трансформации с transition.
  2. JavaScript: События mouseenter и mouseleave, управление классами.
  3. Библиотеки (GSAP): Для сложных анимаций.
8. Структура современных веб-приложений
  • Фронтенд: SPA (React/Vue), HTML/CSS/JS.
  • Бэкенд: API (REST/GraphQL), PHP/Node.
  • База данных: MySQL/PostgreSQL + Redis (кэш).
  • Инфраструктура: Облачные серверы, CDN, CI/CD.
9. Капча (CAPTCHA). Виды и интеграция

Виды: Текстовая, графическая, ReCAPTCHA v2 (галочка), ReCAPTCHA v3 (невидимая, оценка скор).

Интеграция: Вставка JS-скрипта на фронт, проверка токена на бэкенде через API провайдера.

10. AJAX-технологии для создания безопасных веб-приложений

Безопасность AJAX:

  • Всегда проверять CSRF-токены в фоновых запросах.
  • Использовать CORS-заголовки на сервере.
  • Валидировать и экранировать полученные данные.
  • Использовать метод fetch() правильно.

🔑 Управление доступом, Аудит, Блокчейн

11. Управление доступом: определение, задачи, типы

Задачи: Конфиденциальность, Целостность, Доступность.

Типы:

  • DAC (Дискреционное): Владелец управляет правами.
  • MAC (Мандатное): Система на основе меток секретности.
  • RBAC (Ролевое): Права группируются в роли.
  • ABAC (Атрибутное): Динамические правила (время, место).
12. Привилегированные роли и управление привилегированным доступом (PAM)
  • Роли: Суперпользователь (root), админ СУБД, аудитор.
  • Рекомендации: Принцип минимальных привилегий, Just-in-Time доступ, мониторинг сессий.

14. Системы мониторинга и аудита действий пользователей (UAM/UBA)

Функции: Запись сессий, протоколирование, поведенческий анализ.

Решения: Анализ отклонений от нормы (UEBA), блокировка подозрительной активности.

15. Виды надзора за действиями пользователей: плюсы и минусы
  • Полная запись (видео/клавиатура): Дорого, снижает доверие, но полная доказательная база.
  • Логирование событий: Дешево, но нет контекста.
  • DLP-контроль: Проактивная защита, но есть ложные срабатывания.
16. Этическая безопасность

Баланс между защитой активов компании и правом сотрудника на приватную жизнь. Честное информирование о мониторинге и недопустимость чтения личной информации без оснований.

17. NDA и работа с сотрудниками при мониторинге

NDA (Соглашение о неразглашении): Запрещает разглашение конфиденциальных данных (ноу-хау, код) во время и после работы.

При внедрении мониторинга обязательно уведомить сотрудника под подпись.

18. Идеальная система контроля пользователей безопасных приложений

Комбинация RBAC/ABAC, поведенческого анализа (UEBA), контекстного мониторинга и четкой нормативно-документальной базы.

19. Распределённые базы данных и степени прозрачности

Степени прозрачности (скрытости от пользователя):

  • Прозрачность расположения (где лежат?).
  • Прозрачность фрагментации (разбивка по узлам).
  • Прозрачность репликации (копирование).
  • Прозрачность отказов.
20. Блокчейн: определение, функции, пример

Цепочка блоков — распределенный реестр данных, связанных хешированием.

Функции: Децентрализация, неизменность, прозрачность.

Пример: Bitcoin (PoW), передача средств без посредников.

21. Транзакции в блокчейне. Проблемы блокчейна

Майнеры: Проверяют подписи, решают криптозадачу (PoW), получают награду.

Проблемы: Низкая пропускная способность (TPS), высокие энергозатраты (PoW), необратимость ошибок, регулирование.

22. Аудит баз данных. Виды и цели

Цель: Соответствие стандартам и целостность.

Виды: Аудит привилегий, аудит DML/DDL операций, аудит попыток входа.

🔒 Криптография и обфускация

23. Шифрование. Наука и краткая история шифров

От шифра Цезаря и "Энигмы" до современной математической криптографии (DES, AES, RSA).

24. Симметричное, асимметричное шифрование, хэширование
  • Симметричное: Один ключ (AES). Быстро.
  • Асимметричное: Два ключа (RSA, ECC). Медленно, решает проблему распространения.
  • Хэширование: Необратимое преобразование (SHA-256). Для целостности.
25. Алгоритм AES

Симметричный, блочный. Размер блока 128 бит. Ключи: 128, 192 или 256 бит.

Применение: Wi-Fi (WPA2), HTTPS, BitLocker.

Взлом: Устойчив к прямому перебору. Уязвимости в конкретных реализациях (ECB-режим).

26. Алгоритм RSA

Асимметричный. Основан на факторизации чисел.

Применение: Электронная подпись, TLS.

Взлом: Уязвим для квантовых компьютеров (алгоритм Шора).

27. Алгоритм SHA-256

Хеш-функция. Выход 256 бит.

Применение: Биткойн, цифровые подписи.

Взлом: Радужные таблицы (нужна "соль"), брутфорс.

28. Triple DES и протокол Диффи-Хеллмана

3DES: Тройной DES. Медленный, устарел (блок 64 бит).

Diffie-Hellman: Протокол обмена ключами по открытому каналу.

29. Elliptic Curve Cryptography (ECC)

Асимметричное. Математика эллиптических кривых. ECC 256 бит ≈ RSA 3072 бит. Эффективен для мобильных устройств.

30. Обфускация программного кода

Цель: Затруднить анализ кода человеком.

Методы: Переименование переменных (a, b, c), шифрование строк, мертвый код, изменение потока управления (flattening).

🛡️ Угрозы ИБ, Форензика, Защита сетей и Право

31. Триада CIA (Главные элементы ИБ)

Конфиденциальность (C), Целостность (I), Доступность (A).

32. APT-атаки и трояны-бэкдоры

APT: Целенаправленная скрытая долговременная атака (Обнаружение через Threat Hunting).

Бэкдор: Троян для скрытого удаленного доступа.

33. Ботнет и DDoS (Отказ в обслуживании)

Типы DDoS: Канальный (Volumetric), Протокольный (SYN-flood), Прикладной уровень (HTTP-flood).

34. Скрытая загрузка (Drive-by), наборы эксплойтов, метаморфные вирусы
  • Drive-by: Заражение при посещении сайта.
  • Метаморфный вирус: Полная перезапись своего кода при размножении.
35. Инсайдерские угрозы (Типы)

Злонамеренный ("крот"), небрежный (нарушил политику), скомпрометированный (взлом аккаунта).

36. Фишинг, MitM, Сталкинг, Replay-атаки

MitM: Перехват трафика. Replay: Повтор легитимного запроса. Stalkerware: ПО слежки.

37. Шифровальщики, социотехника, кликджекинг

Дроппер: Загружает основную вредоносную нагрузку. Кликджекинг: Невидимые кнопки поверх легитимных.

38. RaaS, RAT и ПНП

RaaS: Вымогатель как услуга. RAT: Троян удаленного доступа. ПНП: Потенциально нежелательное ПО (спам-тулбары).

39. Защита от потери данных (DLP). Компоненты

Агенты на ПК, межсетевые шлюзы, контроль контента (регулярные выражения), блокировка USB и облаков.

40. Обнаружение атак на конечные точки (EDR)

Эволюция антивируса. Мониторинг в реальном времени, поиск IoCs, изоляция хоста.

41. DevSecOps. Ключевые компоненты

Интеграция безопасности на всех этапах CI/CD (SAST, DAST, сканирование зависимостей).

42. Аналитика поведения (UEBA). Три столпа

Сценарии использования (Use Cases) + Источники данных + ML-Аналитика для построения базовой линии поведения (Baseline).

43. Модель нулевого доверия (Zero Trust)

Принцип: Никогда не доверяй, всегда проверяй. Микросегментация и MFA обязательны.

44-47. Компьютерная криминалистика (Форензика)

Процедуры: Создание образа -> Хеширование -> Анализ -> Протокол.

ОС для форензики: Kali Linux, Parrot, Caine.

Техники: Таймлайн-анализ, восстановление данных, поиск стеганографии.

48. Защита баз данных

Шифрование (TDE), контроль доступа, аудит, защита от SQL-инъекций (WAF).

49-51. Конфиденциальная информация и Персональные данные (152-ФЗ)

Персональные данные (ПДн): Любая информация о физ. лице.

152-ФЗ: Согласие на обработку, локализация баз данных РФ. Ответственность: административная и уголовная.

52-53. Безопасность облачных сред

Модели: IaaS, PaaS, SaaS.

Угрозы: Неверные настройки (S3 bucket открыт).

Инструменты: CASB, CSPM, CWPP.

54-55. Аппаратные и автоматизированные системы защиты БД

Примеры: HSM (хранение ключей), SIEM-системы, резервное копирование.

56-58. Протоколы защиты и VPN

VPN (Виртуальная частная сеть): Шифрует канал связи.

Протоколы: OpenVPN (гибкий), WireGuard (современный), IPsec. PPTP: устарел.

Hysteria/VLESS: Протоколы с улучшенной маскировкой и скоростью на нестабильных сетях.

59. Прокси-сервера и их виды
  • HTTP/HTTPS: Для веба.
  • SOCKS5: Более низкий уровень (TCP/UDP).
  • Reverse Proxy: Стоит перед сервером (Cloudflare, Nginx).
60. Инфраструктура фильтрации трафика и DPI

DPI (Deep Packet Inspection): Глубокий анализ содержимого пакетов, а не только заголовков.

Цели: Блокировка запрещенных сайтов, DDoS-защита, приоритизация трафика (QoS), выполнение требований СОРМ.
